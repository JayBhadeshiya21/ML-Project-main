Pipeline Overview

Model Lifecycle: From Data to Decision

This workflow documents the core stages used to generate each cardiovascular risk prediction in real-time.

1

Ingest and Validate Data

Load cardiovascular records from the public dataset and filter clinically inconsistent values — such as negative blood pressure readings or physiologically impossible outliers — before training begins.

2

Preprocess Features

Convert age from days to years, compute BMI from height and weight, apply ordinal encoding to cholesterol and glucose levels, and normalise feature ranges for consistent model input.

3

Train Random Forest

Use an ensemble of decision trees to improve generalisation, reduce overfitting, and handle the non-linear feature interactions that are common in cardiovascular clinical data.

4

Evaluate and Serve

Measure model quality on a held-out test split using accuracy, ROC-AUC, and confusion matrix analysis. The trained model is then deployed via Flask endpoints for real-time inference.
